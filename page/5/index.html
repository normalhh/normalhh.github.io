Toast的工具类--可以取消的Toast

发表于 2015-03-04   |   分类于 JAVA ， 代码   |  

这个工具类主要定义了2种Toast的显示方式，一种是传统的文字Toast，一种是带图片的Toast。而且在使用系统自带的Toast的过程中，总会遇到这样的情况：连续点击的时候，第一个Toast还没有消失，第二个Toast必须要等第一个Toast消失之后才能显示出来，会造成很长时间屏幕上都会有一个Toast在闪烁，下面这个类可以解决这个问题，其实很简单，只要在创建Toast之前先判断一下Toast对象是否为null，不为null则把Toast取消就可以了。PS:这类Toast最好仅用于点击事件中的提示，其他的情况还是要用系统自带的Toast比较合适。

参考代码如下：

package com.tp.insurance.util;

import android.content.Context;
import android.view.Gravity;
import android.view.View;
import android.widget.ImageView;
import android.widget.LinearLayout;
import android.widget.Toast;

/**

  • Created by ex_chenjinghao on 2015/1/28 0028 at 15:06.
    */
    public class ToastUtils {
    private static Toast toast = null;
    public static int LENGTH_LONG = Toast.LENGTH_LONG;
    private static int LENGTH_SHORT = Toast.LENGTH_SHORT;

    public static void TextToast(Context context,CharSequence text,int duration){
if (toast != null) {
    toast.cancel();
}
toast = Toast.makeText(context,text,duration);
toast.setText(text);
toast.setGravity(Gravity.CENTER, 0, 0);
toast.show();

    }

    /**

    • 带图片消息提示
    • @param context
    • @param ImageResourceId
    • @param text
    • @param duration
      */
      public static void ImageToast(Context context,int ImageResourceId,CharSequence text,int duration){
      if(toast != null){
      toast.cancel();
      }
      //创建一个Toast提示消息
      toast = Toast.makeText(context, text, duration);
      //设置Toast提示消息在屏幕上的位置
      toast.setGravity(Gravity.CENTER, 0, 0);
      //获取Toast提示消息里原有的View
      View toastView = toast.getView();
      //创建一个ImageView
      ImageView img = new ImageView(context);
      img.setImageResource(ImageResourceId);
      //创建一个LineLayout容器
      LinearLayout ll = new LinearLayout(context);
      ll.setOrientation(LinearLayout.HORIZONTAL);
      //向LinearLayout中添加ImageView和Toast原有的View
      ll.addView(img);
      ll.addView(toastView);
      //将LineLayout容器设置为toast的View
      toast.setView(ll);
      //显示消息
      toast.show();
      }
      }
       

 

Android中使用到的保存Log信息为txt文档的工具类

发表于 2015-03-04   |   分类于 JAVA ， 代码   |  

在做Android开发过程中，有些时候需要分析一下服务端传递的JSON数据是否有误，而在IDEA中Logcat显示的Log信息有限，并且查看起来相当困难，因此我们需要将Log信息保存成txt文档，这样我们就可以把log信息完整的保存下来。之后使用JSONView工具进行查看就清晰很多了。

这个工具类源码如下：

package com.tp.insurance.util;

import android.content.Context;
import android.os.Environment;

import java.io.*;
import java.text.SimpleDateFormat;
import java.util.Date;

public class Log {

private static Log mLog;
/** 日志保存路径 */
private static final String LOG_SAVE_PATH = "sdcard//Log/";
/** 错误信息保存路径 */
private static final String EXCEPTION_SAVE_PATH = "sdcard//ExceptionLog/";
/** 日志开关 */
private static final boolean LOG_SWITCH = true;

public static Log LogInstance() {
    if (mLog == null) {
        mLog = new Log();
    }
    return mLog;
}

/**
 * 插入日志 logType == 1 数据日志 logType == 2 异常日志
 */
public void saveLog(int logType, String logStr, Context context) {
    if (LOG_SWITCH) {
        if (Environment.getExternalStorageState().equals(
                Environment.MEDIA_MOUNTED)) {
            File file = checkLogFileIsExist(logType);
            if (file == null)
                return;
            FileOutputStream fos = null;
            try {
                fos = new FileOutputStream(file, true);
                fos.write((new Date().toLocaleString() + " " + logStr)
                        .getBytes("UTF-8"));
                fos.write("\r\n\n".getBytes("UTF-8"));
            } catch (FileNotFoundException e) {
                e.printStackTrace();
            } catch (IOException e) {
                e.printStackTrace();
            } finally {
                try {
                    if (fos != null) {
                        fos.close();
                        fos = null;
                    }
                } catch (IOException e) {
                    e.printStackTrace();
                }
                fos = null;
                file = null;
            }
        }
    }
}

/** 检查日志文件是否存在 */
private File checkLogFileIsExist(int logType) {
    File file = null;
    if (logType == 1) {
        file = new File(LOG_SAVE_PATH);
    } else if (logType == 2) {
        file = new File(EXCEPTION_SAVE_PATH);
    }
    if (!file.exists()) {
        file.mkdirs();
    }
    SimpleDateFormat sdf = new SimpleDateFormat("yyyyMMddHHmmss");
    String dateStr = sdf.format(new Date());
    if (logType == 1) {
        file = new File(LOG_SAVE_PATH+ dateStr + ".txt");
    } else if (logType == 2) {
        file = new File(EXCEPTION_SAVE_PATH+ dateStr + ".txt");
    }
    if (!isLogExist(file)) {
        try {
            file.createNewFile();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
    sdf = null;
    return file;
}

/**
 * 检查当天日志文件是否存在
 *
 * @param file
 * @return
 */
private boolean isLogExist(File file) {
    File tempFile = new File(LOG_SAVE_PATH);
    File[] files = tempFile.listFiles();
    for (int i = 0; i &lt; files.length; i++) {
        if (files[0].getName().trim().equalsIgnoreCase(file.getName())) {
            return true;
        }
    }
    return false;
}

/**
 * 打印异常堆栈信息
 *
 * @param e
 * @return
 */
public static String getExceptionStackTrace(Throwable e) {
    if (e != null) {
        StringWriter sw = new StringWriter();
        PrintWriter pw = new PrintWriter(sw);
        e.printStackTrace(pw);
        return sw.toString();
    }
    return "";
}

}
使用方法：

在调用的时候使用：

Log.LogInstance().saveLog(logType, logStr, context);

wordpress加载ajax.googleapis.com页面问题

发表于 2015-02-16   |   分类于 PHP ， 代码   |  

刚刚回到家里，浏览网页的时候看到SAE版的WordPress已经有更新的4.1版本了。在这里感谢这位无私奉献的同学

@Gimhoy

不过更新过之后，就发现自己的博客页面和后台页面打开速度奇慢无比，原因就在于最新版本的WordPress使用了谷歌的字体以及谷歌的ajax，所以在网上百度经验找到一片十分靠谱的文章，在此记录下来以备之后版本升级后修改用。原文地址：http://jingyan.baidu.com/article/86112f135a4b56273697877c.html

修改方法：

  1. 修改 \wp-includes\script-loader.php 文件，查找代码fonts.googleapis.com和ajax.googleapis.com语句（注意：要全部查找到）
  2. 替换fonts.googleapis.com语句为fonts.useso.com
  3. 替换ajax.googleapis.com语句为ajax.useso.com
  4. 转到themes主题的functions.php主题找到http://ajax.googleapis.com/ajax/libs/jquery/1.7.2/jquery.min.js替换为http://ajax.useso.com/ajax/libs/jquery/1.7.2/jquery.min.js（视主题而定，一般国内作者制作的主题都不会使用谷歌的ajax，而老外制作的主题基本上都有谷歌的ajax，如果你使用的是国人制作的主题，可能就不需要修改主题的functions.php，不放心的话可以检查一遍。）
    SAE版本的WordPress的文件修改还需要使用SVN将所有代码Checkout到本地之后进行修改，修改过后必须使用SVN提交到SAE才可以生效。具体的SVN管理SAE代码的教程请查看SAE的帮助文档：

    SVN使用详细说明：http://sae.sina.com.cn/doc/tutorial/code-deploy.html#id2

    在使用SVN过程中，有以下几点需要注意：
